# Linux Log Analysis

## Overview

This project demonstrates how Linux log files can be reviewed to investigate security-related events such as failed login attempts, suspicious authentication activity, and user access history.

Log analysis is an important cybersecurity skill because it helps identify brute-force attempts, unauthorized access, and abnormal behavior on systems.

## Objectives

- Review Linux authentication logs
- Identify failed login attempts
- Understand login history and active sessions
- Practice basic security investigation techniques
- Document findings in a structured format

## Skills Demonstrated

- Linux system administration
- Log analysis
- Security monitoring
- Authentication event investigation
- Command-line troubleshooting

## Files in This Repository

- `ssh-login-analysis.md` - Overview of SSH-related login activity
- `failed-login-investigation.md` - Investigation of failed logins
- `useful-linux-security-commands.md` - Useful commands for log review

## What I Learned

This project improved my understanding of how Linux systems record authentication activity and how logs can be used to detect suspicious behavior and support incident investigation.
